What web does (the recipe, verbatim) 

          

          

            The in-browser result approximates the CLI (wasm encodes AVIF at q80; the
            web CLI uses q85 — DEC-069), so it is close, not byte-identical. Install:
            cargo install crustyimg.
          

        

      


      

        

          100% client-side — no server, no upload, no network call for the conversion.
          
        

        

          crustyimg on GitHub ·
          crustyimg-wasm on npm
        

        

          Honest about the edges: this build encodes AVIF but cannot
          decode it (an .avif you drop is decoded by your browser), and
          its WebP is lossless only (lossy WebP is a C library; this engine is pure
          Rust). Re-encoding an already-lossy file losslessly can come out bigger — which is exactly
          when the never-bigger guard hands your original back.
